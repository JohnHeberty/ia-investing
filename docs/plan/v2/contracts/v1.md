# Contratos canônicos v1

Os contratos em `ia_investing.contracts.v1` são a fronteira compartilhada entre
API, workflows, agents e persistência. Consumidores devem rejeitar campos extras,
datas sem timezone e referências inexistentes.

Mudanças aditivas podem manter `schema_version: "1.0"` quando campos novos forem
opcionais. Remoção, renomeação, alteração semântica ou novo campo obrigatório
exigem um namespace e versão novos. Valores `Decimal` são serializados como texto
em JSON para preservar precisão; IDs usam UUID e timestamps usam ISO 8601.

Fixtures válidas e inválidas ficam em `tests/fixtures/contracts/v1/`. O schema
JSON publicado é gerado diretamente dos modelos Pydantic, nunca mantido à mão.
